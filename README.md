# ESP32-C3 エアコン制御システム

このプロジェクトは、ESP32-C3とSHT30/SHT31（温湿度センサー）を使用し、SwitchBot APIを経由してエアコンを自動制御するシステムです。

セキュリティを保つため、Wi-FiのパスワードやAPIトークンなどの秘密情報はメインプログラム（`main.cpp`）には直接書かず、`secrets.h` という別のファイルに分けて設定します。

---

## 1. `secrets.h` の準備

プロジェクトの `src/` フォルダの中に、新しく `secrets.h` という名前のファイルを作成し、以下のテンプレートをコピー＆ペーストしてください。

```cpp
#ifndef SECRETS_H
#define SECRETS_H

#include <Arduino.h>

// --- Wi-Fi設定 ---
const char* ssid = "YOUR_WIFI_SSID";          // Wi-FiのSSIDを入力
const char* password = "YOUR_WIFI_PASSWORD";  // Wi-Fiのパスワードを入力

// --- SwitchBot設定 ---
const String token = "YOUR_SWITCHBOT_TOKEN";
const String secret = "YOUR_SWITCHBOT_SECRET";

#endif // SECRETS_H
```

---

## 2. Wi-Fi設定の入力

ご自宅のWi-Fiルーターに接続するための情報を入力します。
ルーターの側面や底面に貼られているシールを確認してください。

- **`YOUR_WIFI_SSID`**: ネットワーク名（SSID）に書き換えます。
- **`YOUR_WIFI_PASSWORD`**: 暗号化キー（パスワード）に書き換えます。

> **【注意】**  
> ESP32-C3 は **2.4GHz帯** のWi-Fi（よく `〜-g` や `〜-G` で終わる名前）にのみ対応しています。5GHz帯（`〜-a` や `〜-A`）のSSIDは使用できないため注意してください。

---

## 3. SwitchBot API トークンとシークレットの取得

SwitchBotのサーバーにESP32からアクセスするための「合鍵（トークンとシークレット）」を取得します。
お手持ちのスマートフォンで、普段SwitchBot機器を操作しているアプリを開いてください。

### 取得手順

1. **アプリを開く**
   スマートフォンの「SwitchBot」アプリを開き、右下にある **「プロフィール」** をタップします。
2. **設定画面を開く**
   メニューから **「設定」** をタップします。
3. **開発者オプションを隠しコマンドで出す**
   設定画面の下の方にある **「基本データ」**をタップ
   **「アプリバージョン」**（例：v7.x.x）という文字を **連続で10回タップ** します。
   すると、隠されていた **「開発者向けオプション」** というメニューが新しく出現します。
4. **キーを取得する**
   出現した「開発者向けオプション」をタップします。
   画面に以下の2つの長い文字列が表示されます。
   - **トークン (Token)**
   - **クライアントシークレット (Client Secret)**

これらをコピーして、パソコンに送り、`secrets.h` の該当箇所に貼り付けます。

- **`YOUR_SWITCHBOT_TOKEN`**: 取得した「トークン」に書き換えます。
- **`YOUR_SWITCHBOT_SECRET`**: 取得した「クライアントシークレット」に書き換えます。

---

## 4. セキュリティに関する注意事項

この `secrets.h` には非常に重要な個人情報が含まれています。
プログラムをインターネット（GitHubなど）に公開する場合は、絶対にこのファイルを含めないでください。

本プロジェクトでは `.gitignore` ファイルで `src/secrets.h` を指定しており、誤ってGitにアップロードされないように対策されています。

---

## 5. 初回セットアップ（エアコンの選択と登録）

ESP32にプログラムを書き込み、初めて起動する際、制御対象のエアコン（デバイスID）を本体のROMに記憶させる必要があります。

1. ESP32をPCに接続し、プログラムを書き込みます。
2. シリアルモニタ（ボーレート: `115200`）を開きます。
3. Wi-Fiに接続後、SwitchBotに登録されているデバイス一覧がシリアルモニタに取得・表示されます。
4. リストから制御したいエアコンの番号をシリアルモニタの入力欄に打ち込み、送信（Enter）してください。
5. ESP32が選択されたデバイスIDをROM（不揮発性メモリ）に記憶し、自動的に再起動します。

※ 2回目以降の起動時は、ROMに保存されたデバイスIDを読み込むため、このデバイス選択操作はスキップされ、通常の制御モードに入ります。

---

## 6. 温湿度センサーについて

本システムは **SHT30** および **SHT31** の両方に対応しています。
プログラム内では `Adafruit_SHT31` ライブラリを使用してI2C通信で読み取りを行っています。

- センサーが接続されている場合、通常の処理が開始されると、10秒ごとにシリアルモニタへ現在の温度と湿度を出力します。
- センサーが見つからない場合でも、エラーにはならずエアコン制御の待機状態に入ります（センサー処理のみスキップされます）。

---

## 7. テスト操作

シリアルモニタから直接コマンドを送信することで、エアコンの動作テストが可能です。

- **`1` を送信**: エアコンを ON にします（テスト設定: 26度, 冷房, 風量自動）
- **`0` を送信**: エアコンを OFF にします
